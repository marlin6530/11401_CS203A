# Stack & Queue（堆疊與佇列）

## Stack（堆疊）

### 定義

Stack 是一種**後進先出（LIFO, Last In First Out）**結構。  
元素只能在「頂部」進行插入（push）與移除（pop）。

### 核心操作與複雜度

- push：在頂部加入元素，(O(1))
- pop：移除頂部元素，(O(1))
- top/peek：讀取頂部元素，(O(1))
- isEmpty：檢查是否為空，(O(1))

### 典型應用

- **括號匹配**：編譯器或 IDE 檢查程式碼括號是否成對
- **Undo 功能**：文字編輯器的操作歷史
- **函式呼叫棧**：遞迴呼叫的系統實作方式
- **表達式求值**：中序轉後序、後序運算

---

## Queue（佇列）

### 定義

Queue 是一種**先進先出（FIFO, First In First Out）**結構。  
在「尾端」插入元素，在「前端」移除元素。

### 核心操作與複雜度

- enqueue：在尾端加入元素，(O(1))
- dequeue：從前端移除元素，(O(1))
- front：讀取前端元素，(O(1))
- isEmpty：檢查是否為空，(O(1))

### 典型應用

- **列印佇列**：印表機依送來的順序列印
- **超市排隊**：顧客依到達順序結帳
- **網路封包緩衝**：路由器、交換器的封包緩衝區
- **廣度優先搜尋（BFS）**：圖的層級遍歷

---

## 延伸結構：Deque（雙端佇列）

Deque（Double-Ended Queue）允許在前後兩端皆可插入與移除。  
可以同時模擬 Stack 與 Queue 的行為。

常見用途：

- 回文檢查（頭尾比較）
- 支援從兩端高效推入 / 彈出的緩衝區

---

## Stack vs Queue：何時使用

- **需要記錄「最近的歷史」並且反向回到上一步** → Stack
  - 例：Undo、瀏覽器回上一頁、遞迴展開
- **需要維持「順序處理」** → Queue
  - 例：工作排程、IO 佇列、封包處理、BFS
